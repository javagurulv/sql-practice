К сожалению, начиная с версии MySQL 8.0 и более поздних версий,
MySQL не поддерживает создание ролей напрямую с использованием команды
CREATE ROLE. Вместо этого, в MySQL используется концепция ролей,
называемая "ролями на уровне пользователя" (user-level roles).

Для создания роли на уровне пользователя в MySQL, вам нужно
выполнить следующие шаги:

1. Создайте пользователя: Используйте команду CREATE USER
для создания нового пользователя. Укажите имя пользователя
и пароль для этого пользователя.

Пример создания пользователя в MySQL:
```sql
CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
```
Здесь 'username' - это имя пользователя,
'localhost' - это хост, с которого пользователь может подключаться,
и 'password' - это пароль для пользователя.

2. Назначьте роль пользователю: Используйте команду GRANT
для назначения роли пользователю. Укажите имя роли и имя пользователя.

Пример назначения роли "admin" для пользователя "username":
```sql
GRANT admin TO 'username'@'localhost';
```
Здесь "admin" - это имя роли,
а 'username'@'localhost' - это имя пользователя и хост.

3. Назначьте привилегии роли: Назначьте необходимые привилегии для роли,
которую вы хотите назначить. Используйте команду GRANT для
предоставления привилегий.

Пример назначения привилегии SELECT на таблицу "customers"
для роли "admin":
```sql
GRANT SELECT ON database_name.customers TO admin;
```
Здесь "database_name" - это имя базы данных,
"customers" - это имя таблицы, а "admin" - это имя роли.


Таким образом, в MySQL вы не создаете роли напрямую с помощью
команды CREATE ROLE. Вместо этого, вы создаете пользователя,
назначаете ему роль и назначаете привилегии для этой роли.
Это позволяет вам эффективно управлять доступом пользователей
к данным в базе данных.
